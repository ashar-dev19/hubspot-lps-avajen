Design doc: Convert avajen-landing-a.html to DnD sections

Goal
-----
Make `avajen-landing-a.html` behave like the boilerplate `landing-page.html` by:
- Using a single `{% dnd_area %}` inside the body block.
- Splitting the current hard-coded sections into fixed `section` templates that use `dnd_section`, `dnd_column`, `dnd_row`, and DnD modules.
- Allowing editors to edit only the existing content (texts, images, forms, CTAs), not add fully custom new sections.

High-level structure
--------------------
Page template: `avajen-theme/templates/avajen-landing-a.html`

Body will contain:

{% block body %}
{% dnd_area "main_section"
  label="Main section",
  class="body-container body-container--avajen-landing-a"
%}

  {% include_dnd_partial path="../sections/avajen-header-hero.html" %}
  {% include_dnd_partial path="../sections/avajen-questions.html" %}
  {% include_dnd_partial path="../sections/avajen-free-guide.html" %}
  {% include_dnd_partial path="../sections/avajen-performance-it.html" %}
  {% include_dnd_partial path="../sections/avajen-why-avajen.html" %}
  {% include_dnd_partial path="../sections/avajen-final-cta.html" %}

{% end_dnd_area %}
{% endblock body %}

Notes:
- Each included file will be a `templateType: section` template.
- Set `isAvailableForNewContent: false` in the section front matter so these sections are only used by this landing page.
- Do not add any blank or generic DnD sections; this keeps the page limited to the sections you define.


Section 1 – Header + Hero (with form)
-------------------------------------
Current content (in avajen-landing-a.html):
- Custom `<header class="site-header">` with logo “AVAJEN” + a “Schedule a Review” CTA button.
- Hero section with background image (`hero-bg.jpg`), “Performance-Driven IT” eyebrow, H1 headline, supporting paragraph, two CTA buttons, and a right-hand HubSpot form module (`{% module "hero_form" path="@hubspot/form" %}`).

New section file:
- Path: `avajen-theme/sections/avajen-header-hero.html`
- Front matter:
  - `templateType: section`
  - `label: "AVAJEN header & hero"`
  - `isAvailableForNewContent: false`

DnD layout:
- Wrap the entire hero area in a `dnd_section` with a background image.
- Use two columns:
  - Left column (width ~7–8):
    - Row 1: `@hubspot/rich_text` module for eyebrow + H1 + hero subtitle.
    - Row 2: `@hubspot/rich_text` module for the two CTA buttons (keeping existing classes).
  - Right column (width ~4–5):
    - Row 1: `@hubspot/form` module for the hero form.

Editor capabilities:
- Can edit headline, eyebrow, subtitle text.
- Can edit CTA button text and URLs.
- Can change which form is used in the hero.
- Cannot add new columns or arbitrary extra modules beyond what’s defined.

Constraints and locking down flexibility
----------------------------------------
- No “custom empty section” is added to the DnD area, so editors can only work with the predefined sections listed above.
- Each section is defined as a separate `section` template but marked `isAvailableForNewContent: false` to keep them out of the generic section picker.
- If needed, individual `dnd_module`s (for example logo/header pieces) can be marked as locked so they can’t be moved or deleted, while their text remains editable through the module’s content.
